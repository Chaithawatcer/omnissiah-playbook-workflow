วัตถุประสงค์

1. เพื่อพัฒนาเครื่องมือสำหรับสนับสนุนการรับมือกับภัยคุกคามทางไซเบอร์ โดยสามารถสร้าง Incident Response Playbook ที่มีมาตรฐานเดียวกัน โดยไม่ขึ้นอยู่กับผู้ปฏิบัติงาน
2. ออกแบบระบบ Retrieval-Augmented Generation เพื่อดึง playbook และ procedure ที่เกี่ยวข้อง พัฒนาระบบที่ผสานการค้นหาคืนข้อมูล กับความสามารถในการสร้างข้อความ ของ LLM โดยระบบจะดึงข้อมูล playbook ที่เกี่ยวข้องจากฐานข้อมูลก่อน แล้วจึงนำมาใช้เป็น context ในการ generate คำตอบ
3. ศึกษา LLM สำหรับงาน Cybersecurity มุ่งเน้นการนำ Large Language Model มาประยุกต์ใช้ในบริบทของความปลอดภัยทางไซเบอร์
4. เพื่อออกแบบ Workflow การทำงานของระบบให้สามารถเชื่อมโยงขั้นตอนต่าง ๆ ได้อย่างเป็นระบบ ตั้งแต่การรับข้อมูลจนถึงการแสดงผลลัพธ์
5. เพื่อพัฒนา Prompt Engineering เพื่อควบคุมและปรับปรุงคุณภาพของผลลัพธ์ที่สร้างโดย AI ให้มีความถูกต้องและเหมาะสมกับสถานการณ์
6. เพื่อทดสอบและประเมินประสิทธิภาพของระบบในด้านความถูกต้อง ความสอดคล้อง และความสามารถในการนำไปใช้งานจริงในสถานการณ์จำลอง
ที่มาและความสำคัญ

Playbook คืออะไร: Playbook คู่มือปฏิบัติการแบบทีละขั้นตอนหรือมาตรฐานการปฏิบัติงานที่ถูกเขียนขึ้นมาเพื่อให้ทีมรักษาความปลอดภัยใช้เป็นแนวทางในการรับมือและจัดการกับภัยคุกคามทางไซเบอร์แต่ละประเภทอย่างเป็นระบบ
ความสำคัญของการมี Playbook คือการสร้างมาตรฐานการทำงานที่รัดกุมทำให้มั่นใจได้ว่า ไม่ว่า Alert นั้นจะถูกจัดการโดย Analyst ระดับ Junior ที่เพิ่งเข้ามาใหม่ หรือ Senior ที่มากประสบการณ์ ขั้นตอนการรับมือก็จะออกมาเหมือนกันและได้มาตรฐานเดียวกันเสมอ
ซึ่งปัจจุบันการสร้าง PlayBook ในแต่ล่ะครั้งนั้นใช้ระยะเวลาที่สูงมากเพราะจะต้องหาคนที่มีทักษะที่สูงกับการยืนยันขั้นตอนแต่ล่ะขั้นตอนจึงเป็นสาเหตุที่การจะสร้าง PlayBook ให้ครบและครอบคลุมนั้นจะต้องเสียเวลาเป็นจำนวนมาก
ด้วยเหตุนี้พวกผมเลยคิดระบบ Automation workflow ของ n8n ในการสร้าง Playbook โดยอิงรูปแบบโจมตีจาก Mitre ATT&CK แล้วใช้ AI ในการ generate playbook
วิธีการดำเนินงาน

1.การศึกษาเรื่องของ Framework ต่างๆของ Playbook, Life Cycle ของ Playbook รวมถึงการทำ trouble shooting, identify threat
2. ออกแบบเอกสารสำหรับให้ AI นำมาสร้าง Playbook และ เอกสารสำหรับขั้นตอนต่างๆสำหรับใส่ใน playbook
3.การออกแบบ workflow ให้สามารถใช้งานได้จริงและสอดคล้องกับการทำงาน
4.การศึกษา model AI ต่างๆ ในการช่วย Generate Playbook และ ศึกษาและหา Prompt system เพื่อให้ AI ทำงานตามที่กำหนดไม่ทำนอกเหนือคำสั่งหรือแต่งเติมคำสั่งเพิ่มเติม
5.ทดสอบและทำการยืนยัน Playbook ที่ AI Generate มาตรวจสอบแก้ไขเพื่อพัฒนาการทำงานของ AI ในครั้งถัดไป
ขอบเขตของโครงงาน

1. ระบบรองรับการสร้าง Playbook สำหรับภัยคุกคามทางไซเบอร์ จาก technique ที่พบเจอมาก เช่น phishing
2. ใช้ข้อมูลจาก Framework มาตรฐาน เช่น MITRE ATT&CK และ Playbook ด้าน Cybersecurity เป็นฐานความรู้ในการสร้างและอ้างอิงขั้นตอนการดำเนินงาน
3. ออกแบบและพัฒนา Workflow การทำงานของระบบด้วยเครื่องมือ เช่น n8n เพื่อเชื่อมโยงขั้นตอนต่าง ๆ ตั้งแต่การรับข้อมูล การประมวลผล และการแสดงผลลัพธ์
4. ระบบรับข้อมูลในรูปแบบข้อความ (Text-based input) เช่น alert ที่ผู้ใช้ป้อนเอง
5. พัฒนาระบบโดยใช้ LLM ร่วมกับ RAG เพื่อดึงข้อมูลที่เกี่ยวข้องจากฐานข้อมูลและนำมาสร้าง Playbook อย่างมีบริบท
ประโยชน์ที่คาดว่าจะได้รับ

1. ได้ระบบต้นแบบที่สามารถช่วยสร้าง Incident Response Playbook ได้โดยอัตโนมัติ ลดระยะเวลาและขั้นตอนในการจัดทำเอกสารด้านความปลอดภัย
2. ช่วยให้การรับมือกับภัยคุกคามทางไซเบอร์มีความเป็นมาตรฐานเดียวกัน ลดความผิดพลาดที่อาจเกิดจากผู้ปฏิบัติงาน
3. ช่วยส่งเสริมความเข้าใจเกี่ยวกับ Framework ด้านความปลอดภัย เช่น MITRE ATT&CK และกระบวนการ Incident Response
4. สามารถนำระบบไปต่อยอดเพื่อพัฒนาเป็นเครื่องมือสนับสนุนการทำงานของ Security Operations Center (SOC) ในอนาคต
5. เป็นกรณีศึกษาในการประยุกต์ใช้เทคโนโลยี LLM ร่วมกับ Retrieval-Augmented Generation (RAG) ในงานด้าน Cybersecurity
